Save Files

All the scene data that gets saved is stored in a save file. In the case of MultiScene Persistency, the scene data of all scenes can be saved to a single save file. Save files are encrypted, and can’t be tampered with when opened with Notepad.

Metadata

Sometimes it might be useful to be able to display a list of all the available save files. Most games have a saves menu that does exactly that, and sometimes they also show a save file’s information. By calling SaveManager.GetSavesList() you get the information you need. SaveManager.GetSavesList() returns an array of Metafile. These metafiles contain information (metadata) about the save file they’re associated with. They’re automatically created within the save file itself whenever you create or overwrite an existing save file.

The Metafile class has the following accessible properties:

Metafile

  • Name ( string ): The name of the save file that this metafile is associated with.
  • Date ( string ): Gets you the date of the.
  • LastScene ( string ): This is the name of the scene the player was in during the latest save. Only applicable if the save file is of the Scene File type.

Metafields

You can also supply your own metadata. When saving your game, you can call a different version of SaveManager.SaveScene() or SaveManager.SaveObject() which accepts an array of Metafield.

 

C#

Metafield[] myMetadata =
{ 
    new Metafield("Coins", Player.AmountOfCoins), 
    new Metafield("Character name", Player.CharacterName) 
};
 
SaveManager.SaveScene(mySaveName, myMetadata);

JavaScript

var myMetadata = 
[
    new Metafield("Coins", Player.AmountOfCoins), 
    new Metafield("Character name", Player.CharacterName)
];
 
SaveManager.SaveScene(mySaveName, myMetadata);

 

When creating a new Metafield, you have to pass the name you want to use for it, and the actual value. The value can be an int, float, or string.

File Location

Save files are saved at a persistent location that doesn’t get deleted when you uninstall your game. This location is a folder named ‘Saves’ that’s located at the location that Application.persisentDataPath returns, which is different on every platform. These are the following known locations:

Windows: /Users/<User Name>/AppData/LocalLow/<User Name>/<Product Name>/Saves
Mac: /Users/<User Name>/Library/Caches/<Product Name>/Saves
Linux: /home/[username]/.config/unity3d/[companyname]/[productname]/Saves
Android: /data/data/com.xxx.xxx/files/Saves or SD card: /mnt/sdcard/Android/data/com.xxx.xxx/files/Saves

Forwards Compatibility

Forwards compatbility ensures that you can update your game objects and prefabs over time without breaking existing save files.

Save files are compatible with newer versions of your project. After a save file is loaded, it goes through a check list to see if any game objects that have been saved still exist. If a non-instantiated game object that was saved doesn’t exist in the scene anymore (deleted in the editor by the developer), UniSave will gracefully skip the object. If a game object does still exist but has been modified (components removed or new ones added), it will work fine as well, and will just update the components that were saved and are still present on the game object.

When it comes to instantiated game objects, UniSave will detect if the prefab still exists. If it doesn’t, it will skip instantiating it.

 

If you change the name or data type of a field or property on a component, or move things to a different namespace, existing saved values for them won’t be applied when loading a save file. When there’s a name or type mismatch, they will be skipped during loading. While this will technically not break your save files, it’s just something to keep in mind.

 

Cross-Platform

Save files that were made on one plaform, can be used on another. This means that if you have multiplatform versions of your project, the same save files will work with all of them. The only exception is that save files made on another platform won’t work on iOS, because a different serialization format is being used for that platform.
