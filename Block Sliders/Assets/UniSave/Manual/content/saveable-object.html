Saveable Object

The SaveableObject component is the tool that you use to tell UniSave what should be saved in a scene. Since it doesn’t make sense to save every little bit of data in a scene (this makes your save files larger than necessary, and increases save/load times), SaveableObject allows you to be specific, and only save certain components on a game object.

 

 

When SaveableObject is added to a game object, it will display a list of items that can be selected for saving. Even though the SaveableObject shows three categories (Classes, Components, and Actions), only the Components category is the category that lets you select things. The other two categories are just there to remind you of the default behavior of SaveableObject.

The component list shows all the components that are currently on the game object. When you add new components to the game object, the list will automatically be updated. By selecting a game object, you tell UniSave that it should be saved.

UniSave requires parents of game objects that you’re going to save to have a SaveableObject added to them as well. This is something you don’t have to worry about, as UniSave does this automatically for you whenever you add a SaveableObject to a game object, or when you parent a game object that has a SaveableObject component attached to it.

 

The Ignore SaveScene() option on the SaveableObject gives you the option to let UniSave ignore this SaveableObject when it saves the entire scene. This comes in handy when you’re saving certain SaveableObjects to their own save files manully, and don’t want them to be saved twice. You can read more about saving entire scenes and saving individual game objects on the Saving & Loading page in this manual.

Default Behavior

By default, a SaveableObject saves a game object’s properties like name and tag.  If you don’t intend to save any components but, for example, do want to save the game object’s name, you can add a SaveableObject to the game object and leave things as it is.

Instantation and Destroy events are monitored by SaveableObject. When you’ve instantiated a game object at runtime, UniSave will remember the instantiation if the game object has a SaveableObject on it. This will make sure that this game object will be instantiated (and be put in its pre-saving state) when loading a save file.  In addition, if a game object was placed in the scene in the editor and gets destroyed at runtime, the SaveableObject will registrate this event, and UniSave will make sure this game object stays destroyed.

 

Note that if a SaveableObject is disabled, it will be ignored by UniSave during saving. This could be used as an optimization technique to save specific data only if necessary. For example, if the position of a weapon that’s lying on a table should be saved because it can be moved by the player, you can make it so that its position only gets saved when the object has actually moved from its starting position. The SaveableObject could be enabled when the player touches the item.
