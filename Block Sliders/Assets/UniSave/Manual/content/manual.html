Overview

UniSave is a save system for the Unity game engine. Easy to use, and with some handy features, it’s ideal for saving lots of scene data.

If you want to save lots of game objects in your scenes, Unity has no proper feature for this. While there is the option of using Unity’s built-in PlayerPrefs system, it has certain limitations and is not a viable option for saving entire scenes. Built specifically to fill this gap, UniSave enables you to save your scene’s game objects without having to write any custom code (unlike earlier versions of UniSave), except for a single line of code to either save or load your created save file. UniSave also has extra features that can be used in certain situations.

The UniSave package contains the C# source code (script files), the manual, and an example project. The example project is a basic demonstration of UniSave in action, and shows how it could be integrated into your game.

About the manual

The pages in this manual describe the features of UniSave in detail. Tips and examples of use cases are also given, to give you a better idea of how a certain feature could be used. This manual is written for game developers who are familiar with basic Unity concepts like GameObjects, Components, and Prefabs. Even though some pages might mention a UniSave API call to perform a specific action, you don’t have to be a coder to understand it, and it’s very easy  to follow.

 

Note that this manual gets updated over time with new versions of UniSave. Due to changes and additions, it might not completely correspond with your version if you’re using an older version of UniSave. In that case, it might be better to refer to the offline version of the manual that’s included in the UniSave package.
