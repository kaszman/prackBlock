API

This is a complete list of public methods, properties, and callbacks that you can use as part of the UniSave API. The UniSave API resides in the UniSave namespace. To avoid having to type ‘UniSave’ every time you want to call the API, you can import the library at the top of your script:

 

C#

    using UniSave;

JavaScript

    import UniSave;

SaveManager Class

Static Methods

SaveScene

SaveScene(string saveName)
Saves scene data to a file.

SaveScene(string saveName, Metafield[] metafields)
Saves scene data to a file with custom metadata.

 

SaveObject

SaveObject(string saveName)
Saves the specified game object to a file.

SaveObject(string saveName, Metafield[] metafields)
Saves the specified game object to a file with custom metadata.

 

Load

Load(string saveName)
Loads a save file.

Load(string saveName, string loadScreen)
Loads a save file while displaying a static loading screen image.

Load(string saveName, float fadeInTime, float fadeOutTime)
Loads a save files with a fade-in and fade-out effect.

 

LoadLevel

LoadLevel(string sceneName, bool isPersistent)
Loads a scene. If isPersistent is true, MultiScene Persistency will be used and the current scene will be saved in memory before loading the next one.

LoadLevel(string sceneName, bool isPersistent, string loadScreen)
Loads a scene while displaying a static loading screen image.  If isPersistent is true, MultiScene Persistency will be used and the current scene will be saved in memory before loading the next one. When this is called with isPersistent set to false (no MultiScene Persistency), it is the same as calling Application.LoadLevel(string name), but with a custom loading screen.

LoadLevel(string sceneName, bool isPersistent, float fadeInTime, float fadeOutTime)
Loads a scene with with fade-in and fade-out effect. If isPersistent is true, MultiScene Persistency will be used and the current scene will be saved in memory before loading the next one.

 

Delete

Delete(string saveName)
Deletes a save file from disk in the directory set by CurrentFilePath.

 

SetFilePath

SetFilePath(string filePath)
Changes the current path where saves files are located to the path specified. Paths are relative to the location of ‘(Application.persistentDataPath)/Saves’.
For example, setting the path to “MyDir” will change the current location to  ‘(Application.persistentDataPath)/Saves/MyDir’.

 

GetSavesList

GetSavesList(FileFilterMode filterMode, params Metafield[] customFilters)
Returns a list of all save files found in the directory set by CurrentFilePath. Filter the result by the type of save file and/or the names of custom metafields.

 

ClearCache

ClearCache()
Clears all (persistent) save data that’s stored in memory. This gets called automatically when loading a new save file, but could be called manually in a situation where you can quit the game back to the main menu.

 

DestroyObjectInScene

DestroyObjectInScene(string sceneName, SaveableObject saveableObject)
This only used by UniSave internally to handle game objects moving between scenes. Use DestroyObjectInScene(string sceneName, string tag) instead.

DestroyObjectInScene(string sceneName, string tag)
Destroys an object in another scene that has the specified tag. If multiple objects have the same tag, all of them will be destroyed.

 

Properties

IsSaving
Gets a value indiciating whether UniSave is currently saving.

IsLoading
Gets a value indicating whether UniSave is currently loading.

LastLoadedSceneSaveName
Gets the name of last loaded scene save file.

LastLoadedObjectSaveName
Gets the name of last loaded object save file.

CurrentFilePath
Gets the path to the currently set directory where saves file will be saved to and loaded from.

 

Callbacks

You can use these callbacks in your code just like you would with regular Unity callbacks like Awake() and OnTriggerEnter(). They will get called automatically in your MonoBehaviour scripts.

UniSave_OnLoadingCompleted

UniSave_OnLoadingCompleted()

Gets called when UniSave has finished loading. If you have to perform an action on a saved game object or component as soon as a scene is loaded, you can use this callback instead of Awake or Start, because it gets called when all game objects and components have been updated with saved data, which is after Awake and Start are called.

UniSave_OnSavingCompleted

UniSave_OnSavingCompleted()
Gets called when UniSave has finished saving.

UniSave_OnSavingCompleted(string saveName)
Gets called when UniSave has finished saving data to a file, and returns the name of the file that was saved.
